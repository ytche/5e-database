## 📋 全局规范

### 视觉风格：卡片式

整个小程序采用**卡片式（Card-based）**设计风格，所有内容区块以卡片形式呈现。

**卡片规范**：

| 属性 | 规范 |
|------|------|
| 背景 | 白色或浅灰色卡片，页面背景为浅灰/米白 |
| 圆角 | 12px - 16px |
| 阴影 | 轻微阴影或无边框线分割 |
| 间距 | 卡片之间 12px - 16px 间距 |
| 内边距 | 卡片内部 16px 内边距 |

**卡片层级**：

```
┌─────────────────────────┐  ← 页面（浅灰背景）
│  ┌─────────────────┐    │
│  │                 │    │  ← 卡片1（白底圆角）
│  │                 │    │
│  └─────────────────┘    │
│                         │
│  ┌─────────────────┐    │
│  │                 │    │  ← 卡片2（白底圆角）
│  │                 │    │
│  └─────────────────┘    │
│                         │
│  ┌─────────────────┐    │
│  │  🏠 首页 👤 我的 │    │  ← Tab导航卡片
│  └─────────────────┘    │
└─────────────────────────┘
```

### 底部 Tab 导航

所有页面底部固定显示 Tab 导航栏，以卡片形式呈现：

| Tab | 图标 | 页面 | 说明 |
|-----|------|------|------|
| 首页 | 🏠 | `/index` | 小程序主页 |
| 我的 | 👤 | `/profile` | 个人中心 |

**规则**：
- 所有页面底部都必须包含 Tab 导航卡片
- 当前所在 Tab 高亮显示
- 点击切换时保留页面状态（不刷新）

### 3. 角色创建页面 (`/builder`)

#### 步骤 1：战役设置 (`/builder/campaign`)

**页面说明**：开始创建角色前，配置当前战役的规则版本和启用的扩展规则书（需与 DM 确认）。

**创建流程步骤**：

| 步骤 | 页面 | 名称 | 说明 |
|------|------|------|------|
| 1 | `/builder/campaign` | 战役设置 | 选择规则和扩展书 |
| 2 | `/builder/race` | 选择种族 | 选择种族和亚种 |
| 3 | `/builder/class` | 选择职业 | 选择职业和子职业 |
| 4 | `/builder/abilities` | 分配属性 | 使用购点法分配属性 |
| 5 | `/builder/background` | 选择背景 | 选择角色背景 |
| 6 | `/builder/spells` | 选择法术 | 施法者选择法术（如适用）|
| 7 | `/builder/equipment` | 选择装备 | 选择起始装备 |
| 8 | `/builder/review` | 确认角色 | 预览并确认完成 |

**步骤导航样式**：
- 顶部固定显示步骤指示器
- 当前步骤高亮，已完成步骤可点击返回
- 未开始步骤置灰不可点击
- 格式：`步骤 1/8  战役设置`

#### 步骤 2：选择种族 (`/builder/race`)

**页面说明**：根据战役设置中选择的规则版本和扩展书，展示可用种族列表。用户通过下拉框选择种族，页面展示该种族的详细信息、亚种选择和特性。

**数据加载**：
- 通过 API `GET /data/races` 获取种族列表
- 返回数据包含种族基本信息和亚种列表
- 默认选中第一个种族

**页面结构**：
1. **种族选择下拉框** - 带搜索功能的下拉选择器
2. **亚种选择** - 单选框形式（如有亚种）
3. **基础信息卡片** - 年龄输入、体型、速度、语言选择
4. **种族特性卡片** - 特性列表，分固定和可选两类

**API 数据格式**：

```typescript
interface RaceData {
  index: string;           // 种族ID
  name: string;            // 中文名
  nameEn: string;          // 英文名
  icon: string;            // 图标
  abilityBonuses: AbilityBonus[];  // 属性加成
  speed: number;           // 速度（英尺）
  size: string;            // 体型代码：Small/Medium/Large
  sizeLabel: string;       // 体型显示文本：小型/中型/大型
  age: string;             // 年龄描述文本，显示在年龄输入框下方
  languages: LanguageInfo; // 语言信息
  subraces: Subrace[];     // 亚种列表
  traits: Trait[];         // 种族特性
}

interface LanguageInfo {
  fixed: Language[];       // 固定语言列表（必选）
  optional: Language[];    // 可选语言列表
  optionalCount: number;   // 可选语言选择数量（通常为1）
}

interface Language {
  index: string;           // 语言代码
  name: string;            // 语言中文名
  nameEn: string;          // 语言英文名
}

interface Subrace {
  index: string;
  name: string;
  description: string;
  abilityBonuses: AbilityBonus[];
}

interface Trait {
  index: string;
  name: string;
  description: string;
  optional: boolean;       // 是否可选特性
  options?: TraitOption[]; // 可选特性选项
}

interface TraitOption {
  index: string;
  name: string;
  description: string;
}
```

**交互说明**：
1. **下拉选择种族**：点击下拉框展开列表，可输入搜索，选择后加载该种族数据
2. **亚种选择**：单选框形式，选择后更新基础信息中的属性加成和特性
3. **基础信息卡片**：
   - **年龄**：数字输入框，仅校验为数字，不限制范围，必填；下方显示种族年龄描述（灰色小字）
   - **体型**：从API获取，只读显示（小型/中型/大型）
   - **速度**：从API获取，只读显示（英尺）
   - **语言**：固定语言标签（不可取消）+ 可选语言单选（如有）
4. **特性卡片**：
   - 固定特性：直接展示，无选择按钮
   - 可选特性：内部提供单选按钮供选择
5. **默认状态**：默认选中第一个种族及其第一个亚种

**页面说明**：根据战役设置中选择的规则版本和扩展书，展示可用职业列表。用户通过下拉框选择职业，配置职业技能、熟练项、装备和抉择项。

**数据加载**：
- 通过 API `GET /data/classes` 获取职业列表
- 返回数据包含职业基本信息、生命值信息、熟练项、起始装备、抉择项等

**交互说明**：
1. **下拉选择职业**：点击下拉框展开列表，可输入搜索，选择后加载该职业数据
2. **生命值卡片**：显示生命骰、第一级生命值、后续升级生命值，所有内容从API获取，只读展示
3. **熟练项目卡片**：
   - **护甲/武器**：从API获取，标签展示，只读
   - **工具**：从API获取可选列表，多选标签（如有），必须选够指定数量
   - **豁免**：从API获取，通常是固定的2个，标签展示/选择
   - **技能**：从API获取可选列表，多选标签，必须选够指定数量
4. **装备卡片**：
   - 可能有多条装备选项（如①②③）
   - 每条可能是直接描述（无需选择）或多选（选择N个）
5. **抉择卡片（0个或多个）**：
   - **卡片标题**：从 `cardTitle` 字段获取，如"领域选择"、"战斗风格选择"
   - **显示逻辑**：根据 `choices` 数组动态渲染，有几个就显示几个，无则不显示
   - **每个抉择卡片**：单选框形式，显示选项名称和描述，默认选中第一个选项
6. **默认状态**：默认选中第一个职业，所有需要选择的项目预选足够数量或第一个选项
7. **验证**：必须选择职业，所有需要选择的项目（工具、技能、装备选项、抉择项）都必须完成选择才能进入下一步

#### 步骤 4：分配属性 (`/builder/abilities`)

**页面说明**：根据战役设置，用户使用骰值选取或27购点法分配六项属性值。

**页面结构**：
1. **模式切换页签** - 骰值选取 / 27购点
2. **骰值选取模式** - 掷骰生成数组，选组后分配
3. **27购点模式** - 使用27点预算购买属性值

**API 数据需求**：

```typescript
// 从种族步骤传递过来的数据
interface RaceBonuses {
  [ability: string]: number;  // 如 { str: 2, dex: 0, ... }
}

// 从职业步骤传递过来的数据
interface ClassInfo {
  index: string;
  name: string;
  primaryAbility: string;       // 主属性，如 "int"
  secondaryAbilities?: string[]; // 副属性，如 ["wis", "con"]
}

// 属性分配结果
interface AbilityAllocation {
  mode: 'rolled' | 'point-buy';
  // 骰值模式
  rolledGroup?: number;       // 选择了第几组
  rolledValues?: number[];    // 该组的6个数值
  assignments?: {             // 分配到各属性
    [ability: string]: number;
  };
  // 购点模式
  baseScores?: {              // 购点后的基础值
    [ability: string]: number;
  };
  // 最终结果（两种模式都有）
  finalScores: {
    [ability: string]: number;  // 基础值+种族加成
  };
  modifiers: {
    [ability: string]: number;  // 调整值
  };
}
```

**骰值选取交互说明**：

1. **选择组数**：DND3（3组）或 DND5（5组），默认3组
2. **开始掷骰**：点击后生成指定数量的数组
   - 每组：4d6取3，重复6次
   - 显示每组6个数值，从大到小排列
3. **选择组**：点击"选择此组"，高亮显示
4. **分配到属性**：
   - 每个属性下拉框显示该组6个数值
   - 已选的数值在其他属性中标记为已使用
   - 显示种族加成和最终值/调整值
   - **职业标识**：主属性显示"★"，副属性显示"☆"
     - 如：法师主属性→智力★，副属性→感知☆/体质☆
5. **标准数组快捷**：提供 [15][14][13][12][10][8] 一键选择
6. **验证**：6个属性都分配了值才能下一步

function saveCharacter(character: CharacterReview) {
  // 1. 生成唯一ID
  const id = generateUUID();
  
  // 2. 保存到本地存储（微信小程序 storage）
  wx.setStorage({
    key: `character:${id}`,
    data: character
  });
  
  // 3. 更新角色列表
  const list = wx.getStorageSync('character:list') || [];
  list.push({ id, name: character.name, createdAt: Date.now() });
  wx.setStorageSync('character:list', list);
  
  // 4. 跳转
  wx.navigateTo({ url: `/character/${id}` });
}

**法师特殊机制说明**：

法师采用"**法术书 + 每日准备**"的混合机制：

| 阶段 | 卡片 | 数量 | 更换频率 | 说明 |
|------|------|------|----------|------|
| 抄写法术书 | 📖 | 6个 | 升级时 | 选择6个1环法术长期学习 |
| 每日准备 | 📜 | 3个 | 每日长休 | 从法术书中选择当日可用 |
| 法术位 | - | 2个 | 短休/长休 | 施法次数限制 |

**实际使用流程**：
1. 车卡时：选6个进法术书 → 从中准备3个
2. 升级时：可以更换法术书中的法术
3. 每日长休：可以更换准备的法术
4. 战斗中：只能用"已准备"的法术（3个），但可以用法术位施放任意已准备的法术

**对比其他职业**：
- 法师：法术多（6个在书），每日灵活（3个准备），战术深度高
- 术士：法术少（2个已知），每日固定，但可以用术法点强化
- 牧师：法术中（约4-5个准备），每日灵活，神祇赐予

  proficiency: number
): number {
  return getModifier(abilityScore) + proficiency;
}

# Git 提交规范

> 创建时间：2026-02-16
> 
> 适用范围：D&D 5e SRD 中文翻译项目

---

## 核心要求

### ✅ 必须使用中文提交信息

所有 `git commit` 的 message 必须使用**中文**撰写。

---

## 提交信息格式

```
<类型>(<范围>): <简短描述>

<详细描述（可选）>

<底部信息（可选）>
```

### 类型（Type）

| 类型 | 中文说明 | 使用场景 |
|-----|---------|---------|
| feat | 新功能 | 新增翻译文件 |
| fix | 修复 | 修复翻译错误、术语统一 |
| docs | 文档 | 更新术语表、工作流程等 |
| refactor | 重构 | 调整文件结构（无内容变更） |
| style | 格式 | 代码格式调整（非翻译内容） |
| chore | 琐事 | 构建脚本、配置更新 |

### 范围（Scope）

| 范围 | 说明 |
|-----|------|
| zh | 中文翻译相关 |
| glossary | 术语表更新 |
| workflow | 工作流程更新 |
| review | 审查报告 |

---

## 示例

### 新增翻译文件
```bash
git commit -m "feat(zh): 翻译 Equipment-Categories (983条)

- 翻译 39 个装备分类
- 翻译 944 个装备条目
- 添加 name_en 保留英文原文"
```

### 修复翻译错误
```bash
git commit -m "fix(zh): 统一术语表中的武器名称

- Club: 短棒 → 棍棒
- Greatclub: 大棒 → 巨棒
- Pike: 长矛 → 长枪"
```

### 更新术语表
```bash
git commit -m "docs(glossary): 添加艾恩石系列术语

- 补充 15 种艾恩石标准译名
- 修正 Waterskin: 睡袋 → 水袋"
```
